# 调试功能使用说明

## 🎯 功能概述

为了方便调试签名问题，我在前后端都添加了详细的调试日志输出。

---

## 📡 后端调试日志 (Rust)

### 日志位置
`record-api/src/signature.rs` - `signature_middleware` 函数

### 输出内容
每个请求都会输出以下信息：

```
🔍 [SIGNATURE DEBUG] GET /api/profile
  📋 Headers:
    x-signature: abc123...
    x-timestamp: 1721467114
    x-nonce: xyz789...
    x-app-key: record_app_v2
  📝 Body Hash: e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855
  🔐 Expected Sign String: GET|/api/profile|1721467114|xyz789...|e3b0c44...
✅ [SIGNATURE OK] GET /api/profile   # 或 ❌ [SIGNATURE FAILED]
```

### 查看日志
```bash
cd record-api
cargo run  # 或 RUST_LOG=info cargo run
```

### 2. 调试面板 (UI)

**文件**: `record-app/lib/utils/debug_helper.dart`

#### 使用方法

1. **启用/禁用调试模式**
```dart
// 在 api_service.dart 中
ApiService.debugMode = true;   // 开启（默认）
ApiService.debugMode = false;  // 关闭
```

2. **显示调试面板**
在任何地方调用：
```dart
DebugHelper.showDebugPanel(context);
```

3. **添加浮动按钮**（可选）
在你的页面 Stack 中添加：
```dart
Stack(
  children: [
    // 你的页面内容
    ...
    DebugFloatingButton(),  // 调试浮动按钮
  ],
)
```

#### 调试面板功能
- 显示最近 50 条 API 请求记录
- 每条记录可展开查看详情：
  - URL
  - Headers（包含签名信息）
  - Body
  - Response
- 支持清空日志
- 成功请求显示绿色圆点，失败显示红色

## 🔧 如何关闭调试功能

### 调试完成后，按以下步骤关闭：

#### 1. 前端 - 关闭控制台日志
```dart
// api_service.dart
static bool debugMode = false;  // 改为 false
```

#### 2. 后端 - 注释掉调试日志（可选）
编辑 `record-api/src/signature.rs`，注释或删除以下代码块：
- 第 182-187 行：Header 输出
- 第 225-226 行：Body Hash 输出  
- 第 240 行：成功提示

#### 3. 删除调试面板 UI（可选）
删除或不再引用 `debug_helper.dart`

## 🐛 调试签名问题的步骤

1. **查看前端日志**
   - 运行 Flutter 应用
   - 查看 Android Studio/Xcode 控制台输出
   - 找到 `🔍 ══════` 标记的调试信息
   - 确认 Headers 是否正确生成

2. **查看后端日志**
   - 运行 Rust 服务器
   - 查看终端输出
   - 找到 `🔍 [SIGNATURE DEBUG]` 标记的信息
   - 对比前后端的签名参数是否一致

3. **常见问题排查**

   | 问题 | 可能原因 | 解决方案 |
   |------|---------|---------|
   | Missing headers | 前端没发送签名头 | 检查是否使用 `_signedRequest()` |
   | Invalid timestamp | 时间戳过期 | 检查设备时间是否准确 |
   | Invalid signature | 签名计算不一致 | 对比 sign string 是否相同 |
   | Duplicate nonce | 重放攻击检测 | 正常情况，每次请求应使用新 nonce |

4. **对比关键值**

   前后端应该一致：
   - Method (GET/POST/PUT/DELETE)
   - Path (/api/xxx)
   - Timestamp (Unix 秒)
   - Nonce (随机字符串)
   - Body Hash (SHA256 of body)
   - 最终的 Sign String: `method\|path\|timestamp\|nonce\|bodyHash`

## 📝 示例：完整的调试流程

### 场景：更新用户档案返回 400 错误

#### 步骤 1：查看前端控制台
```
🔍 ════════════════════════════════════════════════════
Method: PUT
URL: https://39.105.113.213:3000/api/profile
Headers:
  X-Signature: a1b2c3d4e5f6...
  X-Timestamp: 1721467114
  X-Nonce: AbCdEfGhIjKlMnOp
  X-App-Key: record_app_v2
Body: {"name":"Test","current_weight_kg":70,...}
```

#### 步骤 2：查看后端日志
```
🔍 [SIGNATURE DEBUG] PUT /api/profile
  📋 Headers:
    x-signature: a1b2c3d4e5f6...     ← 应该与前端一致
    x-timestamp: 1721467114           ← 应该与前端一致
    x-nonce: AbCdEfGhIjKlMnOp        ← 应该与前端一致
    x-app-key: record_app_v2          ← 应该与前端一致
  📝 Body Hash: xxxxx...
  🔐 Expected Sign String: PUT|/api/profile|1721467114|AbCdEf...|xxxxx...
❌ [SIGNATURE FAILED] PUT /api/profile
  Provided Signature: a1b2c3d4e5f6...
  Sign String: PUT|/api/profile|1721467114|AbCdEf...|xxxxx...
```

#### 步骤 3：分析问题
如果 `Provided Signature` 和用 `Sign String` 计算出的结果不同，说明：
- 密钥不一致
- Body hash 计算方式不同
- 编码问题

## ✅ 完成调试后

确认所有 API 都正常工作后：

1. 设置 `ApiService.debugMode = false;`
2. 可选：清理 `signature.rs` 中的调试日志
3. 可选：移除 `DebugFloatingButton`
4. 重新构建 release 版本

```bash
# Flutter
cd record-app && flutter build apk --release

# Rust
cd record-api && cargo build --release
```
